习题 2.22 下列程序虽然合法，但是风格很糟糕。有什么问题呢？怎样改善？
	for(int i = 0; i < 100; ++i)
		//process i

解： 使用100 作为上界，100意义在上下文中也没有体现出来，如果一个程序中多个地方使用了100,以后需要修改，改动量就比较大。
    可以使用一个const 变量来代替100.

习题 2.23  下列哪些语句合法？ 对于那些不合法的，请解释为什么不合法。
	(a) const int buf; //不合法， const变量在定义是必须初始化
	(b) int cnt = 0;   //合法
		const int sz = cnt;
	(c) cnt++;sz++;     //不合法， 修改了sz大小
